## AI 使用方式

### 何时先读本文件

在以下情况下，先读本索引，再深入：

- 任务涉及多个模块（如"给 Todo 加一个指派给用户的功能"）
- 不确定某个功能应该属于哪个模块
- 新增一个业务模块后，需要更新本索引和创建模块文档

### 新增模块的步骤

当你新增一个业务模块时，必须按以下步骤操作：

1. 在 `docs/modules/` 下创建 `xxx-module.md`（参考 [user-module.md](user-module.md) 的格式）
2. 在本文件的「模块列表」中新增一行
3. 如模块涉及新数据表，更新 `docs/database-schema.md`
4. 如模块涉及新 API，更新 `docs/api-contracts/`

### 模块文档的标准格式

每个模块文档应包含以下章节（详见 user-module.md 的示例）：

- **概述** — 一句话说明模块是做什么的
- **职责边界** — 本模块负责什么、不负责什么
- **核心数据模型** — 主要的类型/接口定义
- **Service 接口** — 对外暴露的方法签名
- **依赖关系** — 依赖哪些模块、被哪些模块依赖
- **数据库表** — 关联的数据表（引用 database-schema.md）
- **安全规则** — 模块级别的安全约束
